四种概率/分布相关 Flow 的直觉对比
先看上方插图建立直觉，再看中间的公式做核对，最后看底部对比表做归类。 四种方法都和「概率分布」有关。注意 Gumbel-Softmax 有两个同名但不同的方法： 2016 vanilla 版（Jang/Maddison）是 categorical reparameterization 技巧、 不是 flow；2025 Flow Matching 版（Tang et al. arXiv:2503.17361） 把同一个 softmax interpolant 配上时间相关的温度调度 $\tau(t)$ + ODE 求解， 就是 flow matching。本仓库的 GumbelSoftmaxBridge 实现的是后者。
看图时只需盯住这 3 件事
1. 中间状态是不是始终还能解释成「合法概率分布」。
2. 它在哪种几何空间里走（欧氏 / 单纯形 / 球面）。
3. 它真的是 flow，还是另一类技巧（如 reparameterization）。
Gaussian FM
想象在一张白纸上，从随机点 A 直线走到目标点 B。路径上每个中间点只是普通向量，不一定是合法分布。
Dirichlet
想象在概率三角形里，把均匀撒开的沙子慢慢推向某个角落。整个过程它一直是合法分布。
Fisher
先把概率开方变成球面上的点，再沿球面最短弧线走——这是「分布几何」里的真直线。
Gumbel-Softmax
两个同名方法：2016 vanilla 用固定温度做 reparameterization， 不是 flow；2025 Flow Matching 把温度做成 $\tau(t)$ 调度 + ODE 求解，是 flow。本仓库实现的是 2025 FM 版。

下面四列与上方一一对应。统一采用 3 类例子，目标类别设为第 2 类，即目标 one-hot $\mathbf{e}_2 = [0,\,1,\,0]$。
Gaussian FM
原理公式
状态空间：欧氏 $\mathbb{R}^K$
线性插值路径：
$$z_t = (1-s)\,z_0 + s\,z_1,\quad s\in[0,1]$$
数字 Example
$z_0 = [0.2,\,-1.0,\,0.5]$
$z_1 = \mathbf{e}_2 = [0,\,1,\,0]$，$s = 0.25$
$$z_t = 0.75\,z_0 + 0.25\,z_1 = [0.15,\,-0.50,\,0.375]$$
注意：第 2 项 $-0.50$ 为负，三项之和 $=0.025$，并非合法概率分布。
和其他方法的区别
中间状态只是普通连续向量，不保证非负，也不保证总和为 1。所以它不像「类别概率分配」， 更像在连续编码空间里做 flow。
Dirichlet
原理公式
条件路径：
$$x_t \sim \mathrm{Dir}(\mathbf{1} + t\,\mathbf{e}_c)$$
均值：
$$\mathbb{E}[x_t] = \frac{\boldsymbol{\alpha}}{\alpha_0},\quad \alpha_0 = \sum_i \alpha_i$$
数字 Example
$K = 3,\ t = 4,\ c = 2$
$\boldsymbol{\alpha} = \mathbf{1} + 4\,\mathbf{e}_2 = [1,\,5,\,1],\ \alpha_0 = 7$
$$\mathbb{E}[x_t] = \left[\tfrac{1}{7},\,\tfrac{5}{7},\,\tfrac{1}{7}\right] \approx [0.143,\,0.714,\,0.143]$$
典型采样会接近 $[0.10,\,0.80,\,0.10]$ 这类合法分布。
和其他方法的区别
它从一开始就把状态定义成合法概率分布。问题在于同一个 $t$ 在不同 $K$ 下不再代表 同样尖锐的状态（$\alpha$ 尺度随 $K$ 漂移）。
Fisher
原理公式
开方映射到球面：
$$u = \sqrt{p},\quad \sum_i u_i^2 = 1$$
球面最短弧线（slerp）插值：
$$u_t = \mathrm{slerp}(u_0,\,u_1,\,s),\quad p_t = u_t^2$$
数字 Example
$p_0 = [\tfrac{1}{3},\tfrac{1}{3},\tfrac{1}{3}],\ p_1 = \mathbf{e}_2 = [0,1,0]$
$u_0 = \sqrt{p_0} \approx [0.577,\,0.577,\,0.577]$
$u_1 = \sqrt{p_1} = [0,\,1,\,0]$，取 $s=0.5$
$$u_t \approx [0.325,\,0.888,\,0.325]$$
$$p_t = u_t^2 \approx [0.106,\,0.789,\,0.106]$$
和其他方法的区别
它不是在普通平直空间里走，而是先换到更适合概率分布的几何坐标里（球面）， 再沿那个几何里的最短路径走。
Gumbel-Softmax
原理公式
带温度的 softmax：
$$x_t[j] = \mathrm{softmax}\!\left(\frac{\ell_j + g_j}{\tau}\right)$$
$\tau$ 大 ⇒ 分布更软；$\tau$ 小 ⇒ 分布更尖。
数字 Example
$\boldsymbol{\ell} = [0.3,\,1.1,\,-0.2]$
$\mathbf{g} = [0.1,\,0.0,\,-0.4]$（说明性 noise，非真实抽样）
$\boldsymbol{\ell} + \mathbf{g} = [0.4,\,1.1,\,-0.6]$
$\tau = 1.0:\ x_t \approx [0.296,\,0.595,\,0.109]$
$\tau = 0.3:\ x_t \approx [0.088,\,0.909,\,0.003]$
和其他方法的区别
⚠ 注意区分两个同名方法：
2016 vanilla（Jang/Maddison）： 温度 $\tau$ 是固定/手调的超参，目的是让离散类别在 forward 上「软化」 以便求梯度 — 不是 flow，是 reparameterization。

2025 Flow Matching（Tang et al. arXiv:2503.17361）： 温度做成时间调度 $\tau(t)=\tau_{\max}e^{-\lambda t}$， 整条 $x_t$ 是单纯形上的 conditional path，配条件速度 + ODE 求解 — 是真正的 flow matching。本仓库 GumbelSoftmaxBridge 实现的就是这一版（默认 $\tau_{\max}{=}10$，$\lambda{=}3$，$\beta{=}2$）。

上面公式里的 noise $\mathbf{g}$ 仅为说明性，并非真实从 $\mathrm{Gumbel}(0,1)$ 抽样。
Gumbel-Softmax Flow Matching (Tang 2025) 原理详解
上面那个 panel 只画了一个 softmax 公式。Tang 2025 真正在做的是把同一个 softmax 升级成「单纯形上的连续 path + ODE 求解」。下面 7 个 box 把整套机制 一步步拆开，对应公式编号沿用论文 (arXiv:2503.17361 §4-5)。本仓库 GumbelSoftmaxBridge 严格按这套实现。
0 · 它要解决什么问题
我们想在 离散类别空间(K 个类) 上做生成。普通 Flow Matching 在 $\mathbb{R}^K$ 里做直线插值，但离散类别天然约束在 K-1 维单纯形 $\Delta^{K-1}$ 上 — 中间状态不能「负概率」或「和 ≠ 1」。

已有方法各有问题：
· Dirichlet FM (Stark 2024)：高 K 下有 support shrinkage 病理 (Stark Prop. 1 自己指出的 failure mode)，还要预算 $c$-factor 网格。
· Fisher FM (Davis 2024)：走球面测地线，几何优雅但实现要 sqrt + slerp。

Tang 的洞察：用 Gumbel-Softmax 自身作为单纯形上的 path interpolant — softmax 输出本来就是合法概率分布，温度 $\tau$ 给了一个「噪声 → 尖锐」的 连续插值轴。
1 · 路径 (Eq. 9 — interpolant)
给定目标类别 $k$ (one-hot $e_k$)，时间 $t \in [0, 1]$，中间状态： $$x_t[i] = \mathrm{softmax}_i\!\left(\frac{\delta_{ik} + g_i/\beta}{\tau(t)}\right)$$ 其中：
· $\delta_{ik} = 1$ iff $i = k$ (目标 one-hot)
· $g_i$ 是 i.i.d. Gumbel 噪声，$g_i = -\log(-\log U_i),\ U_i \sim \mathrm{Uniform}(0,1)$
· $\beta$ 是噪声强度 (paper 默认 2.0)
· $\tau(t)$ 是时间相关温度 (见下条)

两端行为：
· $t \to 0$：$\tau$ 大，$\delta$ 项被淹没，$x_0 \approx \mathrm{softmax}(g/\beta\tau_{\max})$ ≈ 接近均匀的「噪声分布」。
· $t \to 1$：$\tau$ 小，$\delta_{ik}/\tau$ 项主导，softmax 把质量全推到第 $k$ 维， $x_1 \to e_k$。

中间任意 $t$，$x_t$ 都是合法单纯形元素 (softmax 输出 sum = 1) — 路径全程合法。
2 · 温度调度 (Eq. 8)
$$\tau(t) = \tau_{\max}\, e^{-\lambda t},\quad t \in [0, 1]$$ paper 默认 $\tau_{\max}=10,\ \lambda=3$。意义：
· 起点 $\tau(0) = 10$ → softmax 几乎均匀。
· 终点 $\tau(1) = 10\,e^{-3} \approx 0.5$ → softmax 已经很尖。

代码里加了 $\tau_{\min}=10^{-2}$ floor 防止数值上 $\tau$ 真的趋 0 (paper 没写，但温度方法的标准防护)。
3 · 条件速度 (Eq. 13)
固定目标 $x_1 = e_k$，沿路径求时间导数化简： $$u_t(x_t \mid x_1=e_k)_i = \frac{\lambda}{\tau(t)}\, x_{t,k}\, (\delta_{ik} - x_{t,i})$$ 直观分解：
· $\delta_{ik} - x_{t,i}$ ≈ 「当前 $x_t$ 距离目标 one-hot 还差多少」
· $\lambda / \tau(t)$ ≈ 「时间步尺度」($\tau$ 越小推进越猛)
· $x_{t,k}$ ≈ 「目标维度当前的概率」(强了大胆推，弱了慢慢来)
4 · 边际速度 (Eq. 12，给定模型 endpoint $x_\theta$)
把 Eq. 13 对所有候选目标 $k$ 用模型预测的 endpoint 分布 $x_\theta$ 加权求和并展开，代数恒等化简得： $$u_t^\theta(x_t)_i = \frac{\lambda}{\tau(t)}\, x_{t,i}\, \bigl(x_{\theta,i} - \langle x_\theta, x_t \rangle\bigr)$$ 注意这是闭式 — 不需要内层循环遍历 K 个候选。本仓库 bridges.py:540-548 的 step() 用的就是这个展开形式， 对 GPU 矢量化友好。
5 · 训练损失 (Eq. 11)
$$\mathcal{L} = -\mathbb{E}\!\left[\log \langle x_\theta(x_t, t),\ x_1\rangle\right]$$ 翻译：模型预测 endpoint 分布 $x_\theta$，对 ground-truth $x_1=e_k$ 求 log， 等价于 endpoint 上的 cross-entropy。

这点很重要：loss = 标准 masked CE，跟 classifier-FM 一致， 跟 Fisher / Dirichlet 用的 endpoint parameterization 也一致 — 切换 bridge 只是换 path / step，loss 契约不变。本仓库 bridges.py:227-251 的 compute_loss 是基类共享的 masked CE，所有 active bridge 共用。
6 · 采样 (推理)
从 $x_0 \sim$ Gumbel-Softmax(uniform target) 开始，Euler ODE 沿 $u_t^\theta$ 积分到 $t=1$： 
for t in linspace(0, 1, num_steps):
    x_θ_pred = model.endpoint_logits(x_t, t).softmax()       # 端点预测
    v = (λ / τ(t)) * x_t * (x_θ_pred - <x_θ_pred, x_t>)     # Eq. 12
    x_t = x_t + dt * v
    x_t = simplex_project(x_t)                                # 防漂移
labels = argmax(x_t)
最后 argmax 拿离散类别。Solver 可换 Heun (二阶 predictor-corrector)， 本仓库默认 Euler。
7 · 为什么对我们的 variable-K 场景合适
1. Softmax 归一化随 K 天然稳定 — 没有 Dirichlet 的 support shrinkage，没有 $c$-factor 预算网格。

2. Loss = masked CE — 跟 Fisher / 任何 endpoint-param 方法接口一致，bridge 可以无痛换 (本仓库 build_bridge('gumbel'|'fisher') 只切 1 个字段)。

3. Variable-K 友好 — padding 维度通过 _masked_softmax 在分母里设 $-10^4$ 屏蔽，strict no-op；batch 内不同 $K_b$ 的样本可以 一起训，padding 不污染梯度。

4. 超参少 — 只有 $\tau_{\max},\ \lambda,\ \beta$ 三个， paper §6.2-6.3 已给推荐默认值 (10 / 3 / 2)。

8 · 「flow」在这里到底是什么意思
一句话定义：flow = 时间连续的 path + 由速度场驱动的 ODE。 一个生成模型称为 flow，意思是它通过求解微分方程 $\frac{dx_t}{dt} = u_t(x_t)$，把样本从源分布 $p_0$ 连续地搬到目标分布 $p_1$。

在 Gumbel-Softmax FM 里，这条 flow 的 5 个组件是：
组件 在 Gumbel-Softmax FM 里是什么
状态空间 单纯形 $\Delta^{K-1}$ — 所有合法 K 类概率分布
源分布 $p_0$ 接近均匀：$x_0 \approx \mathrm{softmax}(g/\beta\tau_{\max})$ ($\tau(0)$ 很大，Gumbel 噪声主导)
目标分布 $p_1$ one-hot 集合 $\{e_1, \ldots, e_K\}$
路径 $x_t$ Eq. 9 的 softmax interpolant — 单纯形里一条从「均匀」流向 $e_k$ 的曲线
速度场 $u_t^\theta$ Eq. 12 的闭式表达 — $x_t$ 在每点的「风向」，推它朝模型预测的端点 $x_\theta$ 走

K = 3 几何直观：把 $\Delta^2$ 想成一个三角形，3 个顶点是 $e_1, e_2, e_3$。
         e_2
          ▲
         / \
        /   \
       /  ●  \    ← t=0: 一团点散在中心附近 (≈ 均匀)
      /   ↓   \
     /    ↓    \   ← 速度场 u_t^θ 推它们朝某个角走
    /     ●     \  ← t=0.5: 已经偏向 e_2

每个样本沿三角形内部的曲线走 — 始终在三角形里 (合法概率分布)， 最后落在某个角 (变成离散类别)。

「flow」体现在：时间连续 / 状态在演化 / 演化由速度场决定 / 整条轨迹合法。
「matching」体现在：训练时不真模拟 ODE，而是让模型预测的速度场 在每个采样的 $(t, x_t)$ 上匹配条件速度场 (paper 证明等价于 endpoint CE，见 box 11)。
9 · 背景知识：ODE 是什么
ODE = Ordinary Differential Equation，常微分方程。形式： $$\frac{dx_t}{dt} = f(t, x_t)$$ 物理直觉：已知速度，求轨迹。给定初值 $x_0$， 任意时刻 $x_t$ 由 ODE 唯一决定 (Picard-Lindelöf 定理)。

数值解 (Euler)：把 $[0, 1]$ 切成 $N$ 步，每步 $\Delta t = 1/N$， 迭代： $$x_{t+\Delta t} = x_t + \Delta t \cdot f(t, x_t)$$ 算 N 次 $f$ 就近似拿到 $x_1$。

在 flow matching 里：$f$ 就是速度场 $u_t^\theta$，$t \in [0, 1]$。 从 source 积分到 target 就生成了一个样本 — 「采样 = 解 ODE」。 本仓库默认 Euler，可换 Heun (二阶 predictor-corrector)。
10 · 「闭式解」(closed-form) vs 数值解
闭式解：有显式公式可以直接代入 $t$ 算出 $x_t$，不需要迭代。
数值解：没有显式公式，必须离散化逐步逼近 (Euler / Heun / RK4)。

Gumbel-Softmax FM 有 3 处「闭式」让训练异常高效：

(a) 路径采样闭式 (Eq. 9)
给 $(t, k, g)$，直接 softmax 一次就得到 $x_t$。训练时不用积分 ODE 来采 $x_t$。 这跟 Dirichlet FM 不一样 — Dirichlet 训练时还要预算 $c$-factor 网格才能采样。

(b) 速度场闭式 (Eq. 12)
$$u_t^\theta(x_t)_i = \frac{\lambda}{\tau(t)}\, x_{t,i}\,(x_{\theta,i} - \langle x_\theta, x_t\rangle)$$ 不需要内层循环遍历 K 个候选目标 — 一行 vectorized 操作就够。

(c) 速度从 endpoint 闭式恢复
模型只输出 endpoint 分布 $x_\theta$，速度场用 (b) 闭式算出。 模型不需要直接学速度。

结论：训练侧 100% 闭式，没有任何 ODE 积分或网格预算。 只有推理时才真的需要数值解 ODE — Euler 多步积分。
11 · 为什么不直接监督速度，要监督 endpoint
两种 parameterization 选择：
方式 模型输出 Loss
Velocity FM 速度 $v_\theta(x_t, t)$ $\|v_\theta - u_t\|^2$ (MSE)
Endpoint FM
(classifier-FM)		 端点分布 $x_\theta(x_t, t)$ $\mathrm{CE}(x_\theta,\, x_1)$

Tang 2025 选 endpoint，4 个原因：

1. 速度尺度爆炸 — Eq. 12 里有 $\lambda/\tau(t)$，$t \to 1$ 时 $\tau(t) \to \tau_{\min}$，速度幅值 $\propto 1/\tau_{\min}$ 暴涨。直接 MSE 监督这个量， loss landscape 数值上很烂。Endpoint $x_\theta$ 永远在 $[0, 1]$ 单纯形内，数值稳定。

2. 任务对齐 — 我们最终要的就是 $\arg\max(x_1) = $ 离散类别。 Endpoint 直接就是类别预测，监督它 = 直接监督任务。速度只是中间量。

3. Loss 共享 — masked CE 跟 Fisher / 普通分类任务接口一致。 Bridge 可以无痛换 — 本仓库 build_bridge('gumbel'|'fisher') 只切 1 个字段， loss 契约不变。

4. 理论等价 — Tang 论文 Eq. 11 证明： $$\mathcal{L} = -\mathbb{E}\!\left[\log \langle x_\theta(x_t, t),\ x_1\rangle\right]$$ 等价于 velocity matching loss 的 endpoint-参数化形式。换句话说， 监督 endpoint 就是在监督速度，但实操上更稳更简洁。

代码侧：bridges.py:227-251 的 compute_loss 是 BaseCategoricalFlowBridge 共享的 masked CE，所有 active bridge 都走它 — 这就是「endpoint parameterization」契约的物化。
