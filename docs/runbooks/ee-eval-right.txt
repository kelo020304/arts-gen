# 0617 EE Eval 正确跑法

本文档是 0617 accepted end-to-end eval 的权威入口。重构后不要再找旧的
`scripts/dev/run_0617_*`；真实逻辑已经搬进 `scripts/eval/`。

## 一句话结论

正确链路是：

```text
rendered multi-view images
  -> SS flow concat 生成 whole-object 64^3 voxel
  -> PromptablePartLatentSegNet 生成 per-part voxel
  -> SLat flow 对 whole object 只跑一次
  -> whole voxel 扣掉所有 part voxel 得到 body_without_parts
  -> 按 body_without_parts / part voxel coords 从 whole SLat 切出各组件
  -> decode_slat 输出 mesh + gaussian
```

关键约束：

- `SS_FLOW_FUSION_MODE=concat`。
- `--slat-token-source live` 是正式口径；`cache` 只允许诊断。
- 每个 object 的 SLat flow 只调用一次，summary 里必须是 `"flow_calls": 1`。
- part 输出来自 whole SLat sparse coords 的子集，不再对每个 part 单独跑 SLat flow。
- 图上必须同时包含 `overall`、去除所有 part 后的 `body_without_parts`、以及每个 part。
- 默认 part-seg ckpt 是 S 架构 `part_promptable_seg_full_S_0618-1`
  step 100000。0706 起后续 ee-eval 默认按这个权重走。
  0706 old-vs-M+CC A/B 结论仍保留：M step30000 + CC 未胜出，不能替换当前 S 默认。

  - `slat_stage.condition.token_source == "live_official_trellis_rgba"`

## 大批量 pair 跑法

v5 packed index 在当前 allowed datasets 下有 4535 个 unique object、38926 个 unique obj-angle
pair。需要超过 4535 个样本时不能继续用 `SAMPLE_SELECTION_UNIT=objects`，应使用 pair 口径：

```bash
OUT_DIR=/mnt/robot-data-lab/jzh/art-gen/ee-eval/0701-38926-1 \
LIMIT=38926 TRAIN_COUNT=38926 HELD_COUNT=0 GPUS=0,1,2,3,4,5,6,7 \
ALLOWED_DATASETS=phyx-verse,realappliance,physx-0511-drawer-door \
SAMPLE_SELECTION_UNIT=pairs \
SLAT_TOKEN_SOURCE=live \
EXTRA_ARGS="--split-json /robot/data-lab/jzh/art-gen/ee-eval/0701-38926-1/_inputs/split_from_packed_index.json --packed-index /robot/data-lab/jzh/art-gen/data/part_promptable_seg_packed_v5/index.json --force-export" \
bash scripts/eval/run_ee_eval.bash
```

pair selection 是 round-robin：先覆盖所有 eligible object 的第一个可用 angle，再覆盖第二个
angle，以此类推。因此前 9000 个 pair 会覆盖全部 4535 个 unique object，而不是连续跑同一个
object 的多个 angle。

```text
├── DATASET__OBJECT__angle_00__mesh.png

```

`_platform_runs/` 里保存 SS 和 part stage 中间产物：

```text
voxel.npz
ss_latent.npy
parts/part_XX_voxel.npz
```

加 `--export-mujoco` 时会额外输出：

```text
DATASET__OBJECT__angle_00__mujoco/
├── DATASET__OBJECT__angle_00.xml
└── assets/*.obj
```

注意：默认 ee-eval 产出的是 mesh/Gaussian preview PNG 和 summary；MuJoCo OBJ/XML 是可选导出。
隐藏面黑色顶点的经典补色后处理可用 `--fill-hidden-vertex-colors` 打开，默认关闭；它要求同时加
`--export-mujoco`，只改导出的组件 OBJ 顶点色，不改几何和核心 eval 指标。
mesh/Gaussian panel 的组件顺序是 `overall`、`body_without_parts`、然后是各个 part。
`body_without_parts` 是 whole-object voxel 扣掉所有 part voxel union 后，从同一次 whole SLat
中按 residual coords 切出来再 decode 的结果。

### MuJoCo R-D 后处理要求

RealAppliance drawer/滑动件交付必须走 R-D MuJoCo 导出，不允许回退到 raw/GT mesh：

- mesh 本身按 decoded SLat mesh / decoded Gaussian baked texture 输出；不要为了方向改写 OBJ 顶点。
- Z-forward 修正在 MJCF root object body 上完成：
  `<body name="object" pos="0 0 0" quat="0.707106781 0.707106781 0 0">`
- RealAppliance `joint=prismatic` 或 `joint_type=B` 的预测 part 在自己的 body 内写 slide joint：
  `axis="0 0 1"`，不要用 `0 1 0` 或 `0 -1 0` 作为最终方案。
- drawer body 内必须有 inertial、slide joint、drawer geom；如果浅蓝/玻璃窗还在 body mesh 里，
  从预测 body textured OBJ 按浅蓝连通分量切出 `assets/drawer_glass/mesh.obj`，并把
  `drawer_glass_visual` 挂到 drawer body 下，确保 qpos 变化时同 body 同运动。
- 写 actuator/keyframe：
  `<position name="a_00_00" joint="j_00_00" ctrlrange="0 0.381732268" ctrllimited="true" kp="5" />`
  和 `<key name="home" qpos="0" />`。
- 删除所有 floor/blue checker 内容；MJCF 不写 `blue_checker` texture/material，不写
  `<geom name="floor" ... />`。
- MuJoCo 3.2.7 兼容：不要写 `inertia="shell"`；默认 visual geom 用 group 0；compiler/statistic：
  `<compiler angle="radian" meshdir="." texturedir="." balanceinertia="true" inertiagrouprange="3 5" />`
  和 `<statistic center="0 0 0" extent="1.6" meansize="0.05" />`。

R-D 验证：

```bash
/home/mi/mujoco-3.2.7/bin/compile path/to/object.xml /tmp/compiled.xml

/opt/venvs/arts-gen/bin/python scripts/eval/post/validate_mujoco_rd.py \
  --xml path/to/object.xml
```

`validate_mujoco_rd.py` 会检查 `nq=1,nv=1,nu=1,njnt=1`、`jnt_axis=[0,0,1]`、
`floor_count=0`、`shell_count=0`、`missing_assets=[]`、glass 和 drawer 是否同 body，
并渲染 `mujoco_qpos_compare.png` 对比 qpos=0 和 `0.381732268`。
如果当前机器没有 `/home/mi/mujoco-3.2.7/bin/compile`，报告中会标记
`compile.available=false`；在有该二进制的机器上会自动执行 compile。

## 常见误区

- 不要用 `--slat-token-source cache` 作为正式指标。cache 只用于诊断。
- 不要把 `SS_FLOW_FUSION_MODE` 改成非 `concat`。
- 不要回退到 `scripts/dev/run_0617_128ee_batch.py`。
- 不要对每个 part 单独调用 SLat flow。正确 summary 里 `flow_calls` 必须等于 1。
- 不要只看 parts；正式图上还必须有 `body_without_parts`，即去除所有 part 后的剩余 body。
- 不要随手把默认 part-seg ckpt 替换成其它训练目录；当前默认是 S0618-1 step100000。M step30000 + CC 已测但未胜出，不能作为默认替换。
