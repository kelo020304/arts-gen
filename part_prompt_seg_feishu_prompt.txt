# Part-Prompt-Seg 项目交付汇报

## 1. 项目背景与目标
- 说明 part-prompt-seg 是 3D 可动部件生成链路中的部件分割模块。
- 上游输入来自多视角 RGB 和 2D part masks / VLM 语义部件提示。
- 目标是输出 64^3 部件级 voxel，再接入后续 SLat 解码生成 per-part mesh / Gaussian。
- 用一句链路描述：
  多视角 RGB + 2D part masks -> SS latent -> Promptable Part Seg -> 64^3 part voxel -> per-part mesh / Gaussian

## 2. 本阶段交付范围
请按“已交付能力”组织，而不是按代码文件罗列。

可写要点：
- 核心模型 PromptablePartLatentSegNet 已接入主链路。
- 支持 z_global [B,8,16,16,16] 作为整体物体 SS latent 输入。
- 支持 4 视角 2D part mask prompt，使用 fg_points prompt encoder 编码为 prompt tokens。
- 模型通过 3D trunk tokens 与 mask prompt cross-attention 完成 2D prompt 到 3D 部件区域的映射。
- 当前主线使用 voxel route：
  - 先预测 16^3 cell mask；
  - 再在 whole occupancy 的 64^3 候选体素上做 part/background 分类；
  - 输出 per-part voxel。
- 已打通后续 SLat 解码，支持生成 mesh / Gaussian 预览产物。

## 4. 本轮 T1 增量能力
这一节重点写“本轮新增优化点”和“为什么做”。

### 4.1 Negative Prompt Channel
事实：
- 训练时基于同一个 dataset_id、obj_id、angle_idx 分组，将其他部件的 2D masks 做 union，构造 negative_masks2d。
- negative masks 通过与正向 prompt 相同的 prompt encoder 编码。
- 编码后池化为 summary，经 zero-init 的 negative_prompt_proj 投入正向 prompt tokens。
- zero-init 设计保证初始状态与旧模型等价。
- smoke 中 zero-init equivalence check 已通过，max_abs=0。

交付价值：
- 帮助模型区分目标部件与相邻/相似部件。
- 降低 part 间串扰和误归属风险。

### 4.2 Boundary-band Hard Supervision
事实：
- 在 16^3 cell mask 上计算边界 band。
- 检测 6-neighbor label change 后做 dilation。
- T1 配置使用 boundary_band_radius=2。
- 开启 boundary hard mining，对边界区域中 top 20% difficult samples 额外加权。
- T1 配置：
  - boundary_weight=2.0
  - boundary_hard_mining=True
  - boundary_hard_mining_topk=0.2
  - boundary_hard_mining_weight=2.0

交付价值：
- 强化部件边界学习。
- 面向相邻部件粘连、边界不清晰的问题做针对性优化。

### 4.3 Structured Voxel Corruption
事实：
- 只扰动输入 full_occ，不扰动 target part voxels。
- 扰动包含：
  - occupied voxel dropout
  - dilation shell voxel addition
  - exterior speckle addition
- T1 配置：
  - drop_prob=0.03
  - shell_prob=0.08
  - speckle_prob=0.0003
- 已生成 voxel corruption 可视化样例，确认逻辑生效。

交付价值：
- 提升模型对 whole occupancy 噪声的鲁棒性。
- 降低推理阶段上游 reconstruction 偏差对 part segmentation 的影响。
